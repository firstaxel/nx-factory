`nx-factory` scaffolds and manages Nx monorepos with shared UI and shared auth packages.

## Install

```bash
# Global install
npm install -g nx-factory-cli

# Or run without installing
npx nx-factory-cli init
```

## Command Overview

| Command | Purpose |
| --- | --- |
| `init` | Bootstrap a new Nx monorepo with shared UI package |
| `add-app` | Add a new app in `apps/` and wire shared UI |
| `add-auth` | Scaffold `packages/auth` with provider templates |
| `add-component` | Add shadcn components to shared UI package |
| `remove-component` | Remove shadcn components from shared UI package |
| `update` | Update installed shadcn components |
| `list` | List shared UI components and usage |
| `doctor` | Run workspace checks and auto-fix some issues |
| `add-lib` | Create a generic shared library in `packages/` |
| `add-storybook` | Set up Storybook for shared UI package |
| `publish` | Build and publish shared UI package |

## Common Flags

Many commands support:

| Flag | Description |
| --- | --- |
| `--dry-run` | Preview planned writes/changes without modifying files |
| `-y`, `--yes` | Skip prompts and use defaults/confirmation |

## init

Initialize a new Nx workspace and scaffold shared UI foundations.

```bash
nx-factory-cli init
nx-factory-cli init --name my-monorepo --pkg-manager pnpm
nx-factory-cli init --yes
nx-factory-cli init --dry-run
```

### Options

| Option | Description |
| --- | --- |
| `-n, --name <n>` | Workspace name |
| `-p, --pkg-manager <manager>` | Package manager: `npm`, `pnpm`, `yarn`, `bun` |
| `-y, --yes` | Use defaults without prompting |
| `--dry-run` | Show planned steps only |

### What It Does

- Creates Nx workspace root and package manager workspace setup
- Creates `apps/` and `packages/`
- Scaffolds shared UI package and Tailwind/shadcn setup
- Optionally scaffolds an example app
- Writes `nx-factory.config.json` at the monorepo root

## add-app

Scaffold a new app and wire it to the shared UI package.

```bash
nx-factory-cli add-app
nx-factory-cli add-app --name dashboard --framework nextjs
nx-factory-cli add-app --name marketing --framework vite
nx-factory-cli add-app --name admin --framework remix --yes
```

### Options

| Option | Description |
| --- | --- |
| `-n, --name <n>` | App name |
| `-f, --framework <framework>` | `nextjs`, `vite`, `remix`, `expo` |
| `-y, --yes` | Use defaults without prompting |
| `--dry-run` | Show planned scaffold steps |

### What It Does

- Scaffolds framework project under `apps/<name>`
- Adds `@workspace/ui` dependency to app package.json
- Patches framework config for shared UI styles
- Patches app tsconfig path aliases

## add-auth

Create `packages/auth` and generate provider-specific server/client/next templates.

```bash
nx-factory-cli add-auth
nx-factory-cli add-auth --provider better-auth
nx-factory-cli add-auth --provider clerk --yes
nx-factory-cli add-auth --dry-run
```

### Options

| Option | Description |
| --- | --- |
| `-p, --provider <provider>` | `clerk`, `better-auth`, `workos` |
| `-a, --app <name>` | Target app in `apps/` (accepted by CLI surface) |
| `-f, --framework <framework>` | Framework hint: `nextjs`, `vite`, `remix`, `expo` (accepted by CLI surface) |
| `-y, --yes` | Use defaults without prompting |
| `--dry-run` | Preview generated files and wiring |

### What It Does

- Generates `packages/auth` with provider-specific files
- Builds exports for:
	- `@workspace/auth/server`
	- `@workspace/auth/client`
	- `@workspace/auth/middleware`
	- `@workspace/auth/next`
- Optionally wires selected app package.json files with `@workspace/auth`

### Framework Usage

- Next apps: import middleware/route helpers from `@workspace/auth/next`
- Vite/Expo apps: primarily use `@workspace/auth/client` and point to an auth host
- Remix apps: use `server` + `client` exports and add framework route wiring

## add-component

Add one or more shadcn components to the shared UI package.

```bash
nx-factory-cli add-component
nx-factory-cli add-component button card dialog
nx-factory-cli add-component data-table --dry-run
```

### Arguments and Options

| Input | Description |
| --- | --- |
| `[components...]` | Component names (optional; interactive when omitted) |
| `--dry-run` | Preview only |

### What It Does

- Adds shadcn component files in shared UI package
- Updates barrel exports so components are importable from `@workspace/ui`

## Troubleshooting

### Could not find monorepo root

Run commands from inside your generated workspace root (the folder containing `nx.json` and `nx-factory.config.json`).

### Runtime import/module errors

Rebuild from the CLI package root:

```bash
npm run build
```

### Config file location

`nx-factory.config.json` is written to the created workspace root during `init`.


## `nx-factory-cli init`

Create a new Nx workspace with `apps/` + `packages/` layout.

```bash
nx-factory-cli init
nx-factory-cli init --name my-monorepo --pkg-manager pnpm
nx-factory-cli init --yes
nx-factory-cli init --dry-run
```

### Options

| Option | Description |
| --- | --- |
| `-n`, `--name <name>` | Workspace name |
| `-p`, `--pkg-manager <manager>` | Package manager (`pnpm`, `npm`, `yarn`, `bun`) |
| `-y`, `--yes` | Skip prompts |
| `--dry-run` | Preview only |

### Creates

- Nx workspace files (`nx.json`, root `package.json`, `tsconfig.base.json`)
- `apps/` and `packages/` folders
- Shared UI package in `packages/ui` (or your chosen package name)
- `nx-factory.config.json` in the workspace root

## Troubleshooting

### Could not find monorepo root

Run commands inside your generated workspace root (the folder with `nx-factory.config.json` and `nx.json`).

### Missing module at runtime

Use Node ESM-compatible imports in generated TS output and run a fresh build:

```bash
npm run build
```

### Config file saved in wrong place

`init` saves `nx-factory.config.json` to the created workspace root.
If you moved folders manually, regenerate or move the config to the monorepo root.
