nx-factory-cli scaffolds and manages Nx monorepos with a shared UI package, auth, and a production-ready TypeScript config hierarchy.

## Install

```bash
# Global install
npm install -g nx-factory-cli

# Run without installing
npx nx-factory-cli init
```

## Command overview

| Command | Purpose |
|---|---|
| `init` | Bootstrap a new Nx monorepo |
| `add-app` | Add an app via its official framework CLI |
| `add-auth` | Scaffold `packages/auth` with Clerk, Better Auth, or WorkOS |
| `add-component` | Add shadcn/ui components to the shared UI package |
| `remove-component` | Remove shadcn/ui components |
| `update` | Update installed shadcn/ui components |
| `list` | List components and their usage across apps |
| `doctor` | Validate workspace health and auto-fix barrel exports |
| `add-lib` | Scaffold a typed shared library |
| `add-storybook` | Set up Storybook for the shared UI package |
| `publish` | Build and publish the shared UI package to npm |
| `migrate` | Migrate an existing workspace to the latest configuration |

## Common flags

All commands that write files support:

| Flag | Description |
|---|---|
| `--dry-run` | Preview planned changes without writing anything |
| `-y`, `--yes` | Skip prompts and use defaults |

## init

Bootstrap a new Nx monorepo with shared UI scaffolding.

```bash
nx-factory-cli init
nx-factory-cli init --name my-monorepo --pkg-manager pnpm
nx-factory-cli init --yes
nx-factory-cli init --dry-run
```

### Options

| Option | Description |
|---|---|
| `-n, --name <n>` | Workspace name |
| `-p, --pkg-manager <manager>` | `npm`, `pnpm`, `yarn`, `bun` |
| `-y, --yes` | Use defaults without prompting |
| `--dry-run` | Show planned steps only |

### What it creates

- `tsconfig.base.json` — NodeNext resolution, strict, global `@scope/*` paths
- `nx-factory.config.json` — workspace metadata
- `packages/ui/` — shadcn/ui + Tailwind v4, exports field, tsconfig extending base
- `apps/example-app/` — (optional) Next.js app pre-wired to `@scope/ui`

## add-app

Scaffold a new app using its official framework CLI, then wire it to `@workspace/ui`.

```bash
nx-factory-cli add-app
nx-factory-cli add-app --name dashboard --framework nextjs
nx-factory-cli add-app --name api --framework vite --yes
```

### Options

| Option | Description |
|---|---|
| `-n, --name <n>` | App name |
| `-f, --framework <framework>` | `nextjs`, `vite`, `remix`, `expo` |
| `-y, --yes` | Skip prompts |
| `--dry-run` | Preview without writing |

### Supported frameworks

| Key | Scaffolder |
|---|---|
| `nextjs` | `create-next-app@latest` |
| `vite` | `create-vite@latest --template react-ts` |
| `remix` | `create-remix@latest` |
| `expo` | `create-expo-app@latest` |

## add-auth

Create `packages/auth` with a full auth setup.

```bash
nx-factory-cli add-auth
nx-factory-cli add-auth --provider clerk
nx-factory-cli add-auth --provider better-auth --app dashboard
nx-factory-cli add-auth --provider workos --framework nextjs
```

### Options

| Option | Description |
|---|---|
| `-a, --app <n>` | Target a specific app |
| `-p, --provider <provider>` | `clerk`, `better-auth`, `workos` |
| `-f, --framework <framework>` | Framework hint (`nextjs`, `vite`, `remix`, `expo`) |
| `-y, --yes` | Skip prompts |
| `--dry-run` | Preview without writing |

### Provider packages installed (framework-aware)

| Provider | Next.js | Vite | Remix | Expo |
|---|---|---|---|---|
| Clerk | `@clerk/nextjs` | `@clerk/react` | `@clerk/remix` | `@clerk/expo` |
| Better Auth | `better-auth` | `better-auth` | `better-auth` | `better-auth` |
| WorkOS | `@workos-inc/authkit-nextjs` + `@workos-inc/node` | `@workos-inc/node` | `@workos-inc/node` | `@workos-inc/node` |

## add-component

Add shadcn/ui components to the shared UI package.

```bash
nx-factory-cli add-component
nx-factory-cli add-component button card dialog input
```

Runs `shadcn@latest add --yes` inside `packages/ui` and updates the barrel export.

---

## remove-component

Remove components from the shared UI package.

```bash
nx-factory-cli remove-component button
nx-factory-cli remove-component tooltip badge --yes
```

Removes component files and cleans the barrel export.

---

## update

Update installed shadcn/ui components to their latest templates.

```bash
nx-factory-cli update               # update all
nx-factory-cli update button card   # update specific
nx-factory-cli update --yes
```

Runs `shadcn@latest add --overwrite`. Local modifications to component files will be overwritten.

---

## list

List installed components and show which apps import each one.

```bash
nx-factory-cli list
```

Also flags components that are installed but missing from the barrel export.

---

## doctor

Validate workspace health and auto-fix barrel export issues.

```bash
nx-factory-cli doctor
```

Checks tsconfig inheritance, path aliases, barrel exports, and `package.json` exports field. Auto-fixes missing barrel exports.

## add-lib

Scaffold a typed shared library in `packages/`.

```bash
nx-factory-cli add-lib
nx-factory-cli add-lib --name analytics --type utils
nx-factory-cli add-lib --name api-client --type api --yes
```

### Options

| Option | Description |
|---|---|
| `-n, --name <n>` | Library name |
| `-t, --type <type>` | `utils`, `hooks`, `config`, `types`, `api` |
| `-y, --yes` | Skip prompts |
| `--dry-run` | Preview without writing |

Prompts for **visibility** — `internal` (workspace only) or `public` (published to npm). Shapes both `package.json` and `tsconfig.json` accordingly.

## add-storybook

Add Storybook to the shared UI package.

```bash
nx-factory-cli add-storybook
nx-factory-cli add-storybook --dry-run
```

Runs `storybook init` and generates starter stories for installed components.

## publish

Build and publish the shared UI package to npm.

```bash
nx-factory-cli publish
nx-factory-cli publish --tag next
nx-factory-cli publish --yes --dry-run
```

### Options

| Option | Description |
|---|---|
| `--tag <tag>` | npm dist-tag (default: `latest`) |
| `-y, --yes` | Skip confirmation |
| `--dry-run` | Preview steps without publishing |

Requires the UI package to have `visibility: public` (no `private: true`).

## migrate

Migrate an existing workspace to the latest nx-factory-cli configuration.

```bash
nx-factory-cli migrate
nx-factory-cli migrate --yes
nx-factory-cli migrate --dry-run
```

### Options

| Option | Description |
|---|---|
| `-y, --yes` | Skip confirmation |
| `--dry-run` | Preview what would change |

### What it migrates

| Issue | Action |
|---|---|
| Missing `tsconfig.base.json` | Creates it with NodeNext, strict, paths |
| Package tsconfigs missing `composite` or using `bundler` resolution | Regenerates via factory |
| App tsconfigs missing `extends` or with `packages/**/*` in `include` | Regenerates via factory |
| Missing `uiPackageVisibility` in config | Prompts and updates `nx-factory.config.json` |
| Missing `exports` field in UI `package.json` | Adds it |

Backs up every changed file as `<file>.migration-backup`.

## Quick workflow

```bash
# 1. Create workspace
npx nx-factory-cli init --name acme --pkg-manager pnpm

# 2. Add apps
cd acme
nx-factory-cli add-app --name web --framework nextjs
nx-factory-cli add-app --name admin --framework vite

# 3. Add auth
nx-factory-cli add-auth --provider better-auth

# 4. Add components
nx-factory-cli add-component button card dialog input label

# 5. Build shared packages
pnpm nx run-many --target=build --projects=@acme/ui,@acme/auth

# 6. Start dev
pnpm nx run-many --target=dev --projects=web,admin
```

## Troubleshooting

### Could not find monorepo root

Run commands from inside the workspace (the folder containing `nx-factory.config.json` and `nx.json`). The CLI walks up the directory tree looking for these files, so running from any subdirectory works.

### Missing .js extension errors at runtime

`NodeNext` module resolution requires `.js` extensions on relative imports in `.ts` source files. This is correct — TypeScript compiles `.ts` → `.js` and Node resolves the `.js` at runtime. Any existing imports without `.js` should be updated.

### TypeScript can't find @workspace/* packages

Ensure `tsconfig.base.json` exists at the workspace root and that all app and package `tsconfig.json` files extend it with `"extends": "../../tsconfig.base.json"`. Run `nx-factory-cli doctor` to diagnose.

### Build order errors (package not built)

The `nx.json` `targetDefaults` sets `build.dependsOn: ["^build"]` so Nx builds dependencies first. If a package is missing, run:

```bash
pnpm nx build @workspace/ui --verbose
```

### Existing workspace missing tsconfig.base.json

Run `nx-factory-cli migrate` — it detects and creates the file.
