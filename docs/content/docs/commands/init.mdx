---
title: init
description: Bootstrap a new Nx monorepo with shared UI scaffolding and a production-ready TypeScript config hierarchy.
---

## Usage

```bash
nx-factory-cli init
nx-factory-cli init --name my-workspace --pkg-manager pnpm
nx-factory-cli init --yes
nx-factory-cli init --dry-run
```

## Options

| Option | Description |
|---|---|
| `-n, --name <n>` | Workspace name (lowercase, dashes) |
| `-p, --pkg-manager <manager>` | `npm`, `pnpm`, `yarn`, `bun` |
| `-y, --yes` | Skip all prompts, use defaults |
| `--dry-run` | Preview steps without writing anything |

## Interactive prompts

When run without `--yes`, the CLI asks:

1. **Workspace name** — becomes the npm scope (e.g. `my-app` → `@my-app/ui`)
2. **Package manager** — detected from the environment if possible
3. **UI package name** — default `ui`, lives at `packages/ui`
4. **UI package visibility** — `internal` (private, workspace only) or `public` (will be published to npm)
5. **Base color theme** — `neutral`, `zinc`, `slate`, `stone`, `gray` (sets shadcn/ui CSS tokens)
6. **Pre-install shadcn components** — checkbox selection
7. **Scaffold example Next.js app** — yes/no

## What gets created

```
<workspace-name>/
├── tsconfig.base.json          ← NodeNext, strict, @scope/* paths
├── nx-factory.config.json      ← workspace metadata
├── nx.json                     ← Nx config with build caching
├── package.json                ← workspaces: ["packages/*", "apps/*"]
├── pnpm-workspace.yaml         ← (pnpm only)
├── packages/
│   └── ui/
│       ├── package.json        ← exports field, private/public based on visibility
│       ├── tsconfig.json       ← extends ../../tsconfig.base.json, composite: true
│       ├── components.json     ← shadcn/ui config
│       ├── index.tsx           ← barrel exports
│       ├── lib/utils.ts        ← cn() helper
│       └── styles/globals.css  ← Tailwind v4 + shadcn design tokens
└── apps/
    └── example-app/            ← (if selected) Next.js, pre-wired to @scope/ui
```

## The tsconfig hierarchy

`init` writes a `tsconfig.base.json` at the workspace root that every package and app extends:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "baseUrl": ".",
    "paths": {
      "@my-app/*": ["./packages/*/index.ts"]
    }
  }
}
```

Packages only declare what they uniquely need (`jsx`, `outDir`, `composite`). Apps only declare what they uniquely need (`noEmit`, framework plugins). Nothing is repeated.

## Internal vs public UI package

| | `internal` | `public` |
|---|---|---|
| `private: true` | ✅ | ❌ |
| `exports` field | ✅ | ✅ |
| `files` + `publishConfig` | ❌ | ✅ |
| `stripInternal` in tsconfig | ❌ | ✅ |

This can be changed later by running `nx-factory-cli migrate`.
