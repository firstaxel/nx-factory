---
title: publish
description: Build and publish the shared UI package to npm.
---

## Usage

```bash
nx-factory-cli publish
nx-factory-cli publish --tag next
nx-factory-cli publish --yes
nx-factory-cli publish --dry-run
```

## Options

| Option | Description |
|---|---|
| `--tag <tag>` | npm dist-tag (default: `latest`) |
| `-y, --yes` | Skip confirmation |
| `--dry-run` | Preview steps without publishing |

## Requirements

The UI package must have `visibility: public` (no `private: true` in `package.json`). If you initialised the workspace with `internal` visibility, update it manually or re-run `nx-factory-cli migrate`.

## What it does

1. Runs `tsc -p tsconfig.json` to build `dist/`
2. Confirms the `files` field is set to prevent publishing unnecessary files
3. Runs `npm publish --tag <tag>` from inside `packages/ui`
